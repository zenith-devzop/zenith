## 🔧 Common Issues and Fixes

### ❌ "Failed to Attach" on Injection
**Cause:** Roblox updated after your current Zenith build, or antivirus is blocking the loader.
**Fix:** Restart Zenith — the auto-updater pulls the latest patch automatically. If still failing, add the Zenith folder to your antivirus exclusion list and relaunch as Administrator.

---

### ❌ Script Runs but Has No Effect
**Cause:** The script is outdated, targets a patched game mechanic, or uses a deprecated API.
**Fix:** Test a simple Script Hub script to confirm the executor is working. If Hub scripts execute normally, find an updated version of your script.

---

### ❌ Zenith Crashes Immediately on Windows Launch
**Cause:** Missing .NET 6.0 runtime or outdated Visual C++ Redistributable.
**Fix:** Install the latest Microsoft .NET Runtime (6.0+) and Visual C++ Redistributable 2022 (x64), then relaunch as Administrator.

---

### ❌ Android APK Won't Install
**Cause:** "Install Unknown Apps" not enabled, or insufficient device storage.
**Fix:** Settings → Security → enable Install Unknown Apps for your file manager. Ensure at least 200 MB of free internal storage before installing.

---

### ❌ macOS "Cannot Be Opened" Error
**Cause:** macOS security settings blocking the application.
**Fix:** Go to **System Preferences → Security & Privacy → General** and click **Open Anyway**. For macOS 13+, you may need to right-click the app and select **Open** to bypass Gatekeeper.

---

### ❌ Aimbot Not Tracking Targets
**Cause:** Camera aim mode may be selected or FOV radius may be too low.
**Fix:** Switch to **Mouse Aim** mode in the aimbot settings panel. Increase the FOV radius slider and ensure the correct bone target is selected for your game.

---

### ❌ Script Hub Shows Blank
**Cause:** Internet connection issue or Hub servers temporarily under maintenance.
**Fix:** Check your connection. If online, wait 5–10 minutes and reopen the Hub tab. Maintenance windows are announced in real time in the `#status` channel of the **zenith executor discord**.
